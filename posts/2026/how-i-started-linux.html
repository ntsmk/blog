Here is how I started using Linux on my personal computers

I have installed Linux Mint Xfce on my old Macbook Air beginning of February, 2026. It was my first time to install different OS on my computer. Also, first time to install Linux too. It has been about a month now. And I fell in love with Linux.

How I started my relationship with Linux

One day, my friend said: "Linux is faster than Windows. You should try." His words were stuck with me. Before that, I have never used linux on my personal computer. Although, in the past, I have ran Linux on local VM, AWS EC2, and Raspberry Pi as a learning purpose. But it was more like Linux serve experience. Installing Linux desktop completely on the actual laptop? I felt it is a whole different level.

And shortly after, out of curiosity, I started searching about Linux more. I started seeing more articles basically suggesting to switch to Linux from Windows because how Microsoft nowadays operates. Saying Windows computers are full of bloat which makes the computer slower, a lot of prvacy concerns from telemetry. "Windows treats you as a product, Linux treats you as an admin" I don't know how many times I saw this quotes. It got me more curious about the OS.

Probably the timing was also right. Microsoft ended Windows 10 support on 2025 October. That means you either go upgarade to Windows 11 or go switch to different OS, which is MacOS or Linux. But what if you want to stick with Windows but the computer is old enough not to be able to upgrade to Windows 11?

If you want to stick with Windows, you will need to go buy newer laptop that has Windows 11 already or use specific tool to force Windows 11 upgrade on the old laptop(Which I acually did on the old free Windows laptop I got from work, it was a good experimentation)

Besides the old free laptop I got from work, I already had my Macbook Air from 2015. At this moment, I could not install the latest macOS on the computer anymore because simply 2015 model does not meet the latest OS requirements. And unlike the old Windows laptop which I upgraded RAM to 16GB, this macbook air has soldiered RAM, meaning I cannot upgrade RAM, this macbook air RAM will be stuck with 4GB forever. Even if I use specific tool to force to install new OS to this macbook, simply 4GB RAM can't handle it.

So on random weekend in February, after transfered my all document data on the mac to USB as a backup, finally I decided to try Mint xfce edition live USB on my 11 years old macbook air.

First issue: Wifi not connecting

It was my first time to try Linux live USB in general. I have done booting up from USB before but it was only for Windows iso file. Before booting up, I already asked Gemini what is going to happen, and I was already aware of the issue I will more likely encouter: wifi connecting issue.

It booted up, Mint screen showed up. How it looked was more than polish and modern than I thought. I liked the design. And as I expected, when I clicked the internet connection icon, it was not showing my home wifi SSID. I searched Driver Manager on the seach bar. When I used my smartphone hotspot to get internet for this Mac, the Driver Manager installed Broadcom Wireless drivers, and it started showing my home wifi SSID. I was successfully able to connect to internet while trying live US version.

Besides the wifi connection issue, everything seemed fine. Touchpad works. Scrolling websites works. My biggest concern was wifi connection issue. So after sorting out this issue via Driver Manager, I felt releief and finally decided to wipe the storage completely and install the OS.

Second issue: Newest Kernel is not loaded on the computer

After installing the OS permanently, after creating username and password stuff, I encountered much more difficult problem than Wifi conneting issue. Once I finished the username creation, language slection, timezone etc, I also ran sudo apt update && sudo apt upgrade. I restarted the computer. And it did not load the OS. I was locked out, the real troubleshooting has started.

It seemed like after updating, the newest kernel kicked in, but the compatibility between this kernel and this 2015 Macbook Air was not great, and it ended up struggling with loading the OS.

As a solution for this, first I had to enter the GRUB menu before loading OS. And in the menu, luckily I was able to choose different kernel version. Since the latest kernel version was having an issue to load, so I choose the sightly older version. Was able to successfully load OS, going back to login screen.
